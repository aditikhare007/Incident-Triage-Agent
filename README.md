# 🚨 incident-triage-openenv

**Incident Triage Agent — OpenEnv Evaluation Demo**

This project evaluates how an AI agent performs **incident triage under strict operational constraints** inside a controlled **OpenEnv**.

The agent is assessed on its ability to **interpret incident signals, prioritize correctly, request clarification when needed, and terminate properly**—without performing remediation or autonomous actions.

This is an **evaluation and control environment**, not an incident resolution system.

---

## Evaluation Focus

The agent is evaluated step-by-step on:

- **Signal interpretation**  
  Understanding alerts, logs, and incident metadata

- **Triage correctness**  
  Severity, urgency, and impact classification

- **Clarification behavior**  
  Asking precise, minimal follow-up questions when inputs are insufficient

- **Constraint adherence**  
  Operating strictly within defined scope and rules

- **Proper termination**  
  Ending execution cleanly once triage is complete

---

## What This Is

- A **controlled incident triage evaluation environment**
- A **rule-bound OpenEnv agent demo**
- A **deterministic, inspection-focused system**
- A **trust and reliability assessment layer** for agent behavior

---

## What This Is Not

- An incident remediation system  
- An autonomous operations agent  
- A chatbot  
- A free-form reasoning engine  

The emphasis is on **decision discipline and correctness**, not execution.
